# Documentation

Virtual Table is a browser-based virtual tabletop — a real-time physics sandbox where players spawn boards, dice, cards, tokens, and decks, with optional custom games authored in TypeScript. The documentation here is split by topic; pick the entry point that matches what you want to do.

## Read me first

- [Getting Started](./getting-started.md) — install, run, create or join a room. **For everyone.**

## Playing

- [Controls](./controls.md) — camera, tools, context menu, hand panel, claiming a seat. **For players.**

## Hosting and authoring games

- [Hosting](./hosting.md) — host action bar, editor panel, save/load, asset manager, history, kick/ban. **For room creators.**
- [Scripting](./scripting.md) — author a custom game by extending `Game`, lifecycle hooks, the `scene` API, the entity facade, sandboxing. **For game authors.**

## Working on the codebase

- [Architecture](./architecture.md) — repository layout, entity/component model, networking, scripting host, host/guest authority. **For developers.**
- [Contributing](./contributing.md) — dev setup, scripts, test layout, where to add a component / spawnable / script API / tool. **For developers.**
